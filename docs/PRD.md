# Product Requirements Document (PRD)
## GoBuddy - Intelligent Travel Planning System

### 0.1 เป้าหมาย (Goals)

ผู้ใช้สร้างแผนทริปได้เร็ว (minutes) พร้อมงบประมาณ

จองได้ครบในระบบเดียว (เที่ยวบิน/รถ/โรงแรม/กิจกรรม) หรือผ่าน partner

ระหว่างทริป ระบบปรับแผนตามเหตุการณ์จริง (ฝน, ตั๋วเต็ม, รถดีเลย์, สถานที่ปิด) พร้อมทางเลือกและผลกระทบด้านเวลา/งบ

### 0.2 กลุ่มผู้ใช้ (User Groups)

**Traveler**: เดินทางคนเดียว/คู่/ครอบครัว/กลุ่มเพื่อน

**Trip Organizer**: คนจัดทริปให้ทีม/กลุ่ม

**Admin/Support**: จัดการคอนเทนต์/พาร์ทเนอร์/เคสผิดพลาด

### 0.3 Key KPIs

- **Time-to-plan** (สร้างแผนครั้งแรก) < 10 นาที
- **Plan-to-book conversion %**
- **Replan success rate** (แก้แผนแล้วทำต่อได้จริง) %
- **NPS/CSAT** หลังทริป
- **Booking failure rate** / **Refund handling time**

### 1) ขอบเขต (Scope)

#### In-scope (ต้องมี)

- สร้างทริป (เมือง/วันที่/จำนวนคน/งบ/สไตล์)
- สร้าง itinerary รายวัน + เส้นทาง + เวลาเดินทาง + ค่าใช้จ่ายคาดการณ์
- จองครบ: โรงแรม/กิจกรรม + (Phase ถัดไป: ตั๋วเดินทาง)
- Real-time monitoring: อากาศ/การปิดทำการ/ตั๋วเต็ม/ความหนาแน่น (ตามข้อมูลที่เชื่อมได้)
- Re-planning: แนะนำแผนใหม่ + เปรียบเทียบผลกระทบ + apply to itinerary
- เอกสารทริป: ตาราง, map view, checklist, แชร์ให้เพื่อนร่วมทริป

#### Out-of-scope (ระบุไว้กันบาน)

- การทำราคา dynamic ของตัวเอง (ทำ affiliate/partner ก่อน)
- ระบบ loyalty/points เต็มรูปแบบ (ทำภายหลัง)
- การรับประกันเที่ยวบิน/ประกันการเดินทาง (ทำเป็น integration)
